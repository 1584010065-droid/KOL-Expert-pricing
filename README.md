# KOL 达人核价 MVP Demo

这是基于 PRD `doc/prd/KOL_达人核价_AI功能需求文档v1.1.pdf` 实现的本地 Demo，用于产品演示、流程验证和后续研发团队生产化评估。

当前版本不是生产系统。它使用用户上传的 CSV 作为临时达人样本池，不接入真实达人库、报价库或投放系统。

## 当前进度

已完成一个可运行的多页面 MVP：

1. 上传 CSV
2. 确认字段映射
3. 生成已匹配达人列表
4. 在列表页点击 `AI 核价`
5. 进入单个达人详情页查看核价结果
6. 导出单次核价 JSON

本地服务地址：

```text
http://localhost:5173/
```

## 页面流程

### 1. 上传页

入口：`/upload`

能力：

- 上传 CSV 文件
- 读取表头和达人行数
- 自动识别 PRD 字段
- 统计冗余字段数量
- 提示缺失 P0 字段
- 进入字段映射确认页

说明：

- 冗余字段不会参与 MVP 核价计算。
- 当前 CSV 不会上传到外部数据库，只在浏览器本地状态中使用。

### 2. 字段映射页

入口：`/mapping`

能力：

- 按 PRD 6.1.2 字段清单展示字段映射
- 自动匹配 CSV 表头
- 支持人工修正映射
- P0 字段缺失时禁止进入下一步

当前字段口径以 PRD 截图中的 27 个字段为准。

### 3. 已匹配达人列表页

入口：`/creators`

“已匹配达人”的定义：

```text
已完成字段校验，并且能基于当前 CSV 样本池构建同类达人对标池的达人
```

能力：

- 按内容形式筛选：图文 / 视频
- 按投放目标筛选：曝光 / 种草 / 转化
- 按粉丝量级、一级账号类型筛选
- 按昵称或 uid 搜索
- 列表页直接点击 `AI 核价`

注意：

- 这里的“匹配”不是行业匹配，也不是 AI 匹配。
- 对标池只来自本次上传 CSV，不代表全平台市场价格。

### 4. 单个达人详情页

入口：`/creators/:uid`

能力：

- 展示达人基础信息
- 输入核价任务参数
- 展示系统对标池数据
- 调用真实模型生成核价结果
- 展示核心结论、结构化依据、风险提示和话术
- 导出本次任务 JSON

核价任务参数：

- 平台：当前固定为小红书
- 内容形式：图文 / 视频
- 投放目标：曝光 / 种草 / 转化
- 当前报价：默认读取对应内容形式价格，可人工覆盖
- 投放行业或品类关键词

## PRD 字段口径

MVP 只读取以下字段，其他字段保留在原始行中，但不参与当前核价计算：

| 字段类别 | 字段 |
| --- | --- |
| 达人身份 | uid、昵称 |
| 账号分类 | 一级账号类型、二级账号类型 |
| 粉丝画像 | 粉丝数、粉丝量级 |
| 粉丝质量 | 真粉率 |
| 综合价值 | 传播价值、转化价值 |
| 内容表现 | 爆文率、平均互动量 |
| 互动结构 | 平均点赞数、平均评论数、平均收藏数、互动粉比指数、互动量稳定性指数 |
| 图文表现 | 图文曝光中位数、图文价格、图文预估CPM、图文预估CPE |
| 视频表现 | 视频曝光中位数、视频价格、视频预估CPM、视频预估CPE |
| 商单表现 | 商单发文数、商单平均互动量、商单互动系数 |

## 对标池规则

系统先完成对标池计算，再将结果发送给模型。模型不直接读取全量 CSV 自行计算市场分位。

当前实现：

1. 一级对标池：内容形式 + 粉丝量级 + 一级账号类型 + 二级账号类型
2. 二级对标池：内容形式 + 粉丝量级 + 一级账号类型
3. 三级对标池：内容形式 + 粉丝量级

统计指标：

- 样本量
- 均价
- P25
- P50
- P75
- 最低价
- 最高价
- 当前报价位置
- 当前报价相对 P50 的偏差

报价偏差计算：

```text
(当前报价 - P50 基准价) / P50 基准价
```

页面显示为百分比。例如 `0.56` 显示为 `56%`。

已修复一次模型返回百分比口径问题：如果模型返回 `55.55`，前端会按 `55.55%` 显示；新请求会优先使用系统计算出的偏差值。

## 模型接口

当前使用 DeepSeek OpenAI SDK 兼容接口。

配置文件：

```text
.env
.env.example
```

当前环境变量：

```text
DEEPSEEK_API_KEY=你的 DeepSeek API Key
DEEPSEEK_MODEL=deepseek-v4-pro
```

后端入口：

```text
server/index.js
```

前端不会直接持有 API Key。浏览器请求本地接口：

```text
POST /api/price-check
```

后端再调用：

```text
https://api.deepseek.com/chat/completions
```

模型参数：

```js
{
  model: "deepseek-v4-pro",
  reasoning_effort: "high",
  stream: false,
  extra_body: {
    thinking: { type: "enabled" }
  }
}
```

## 当前 Demo 边界

当前版本可以完成：

- CSV 导入
- 字段映射确认
- 使用当前 CSV 作为达人样本池
- 构建同类达人对标池
- 计算价格分位和报价偏差
- 单达人真实模型核价
- 结构化展示模型结果
- 导出单次核价 JSON

当前版本不能完成：

- 接入真实达人数据库
- 接入历史报价趋势
- 接入评论语义分析
- 接入点击、加购、成交、GMV 等后链路数据
- 批量核价任务队列
- 用户权限和操作审计
- 与公司内部系统打通
- 生产级异常监控、限流和重试

页面上应持续提示：

```text
当前核价基于本次上传 CSV 中的达人样本计算，不代表全平台市场价格。
```

## 给研发团队的关键说明

这个 Demo 的核心价值不是前端样式，而是验证以下业务链路：

```text
CSV 数据源
→ 字段标准化
→ 同类达人对标池
→ 系统计算价格分布
→ 生成单达人核价上下文
→ 模型输出结构化核价结果
→ 前端展示可解释结论
```

生产版本应保留这条链路，但需要把数据、计算、模型调用和日志全部迁移到服务端，以保证安全性、稳定性和可审计性。
